# КасаPOS — облік продажів для магазину

Веб-додаток для обліку продажів: екран продажу у стилі інтернет-магазину, менеджмент товарів
з фото, склад, статистика та історія продажів. Next.js 14 (App Router) + React + TypeScript +
Tailwind CSS + Prisma + PostgreSQL. Адаптивний дизайн (мобільний і десктоп), PWA (можна
встановити на телефон як застосунок).

## Функції

- **Продаж** (`/`) — вибір товарів як в інтернет-магазині, кошик з кількістю, пошук, фільтр за
  категоріями, сканування штрих-коду камерою телефону.
- **Оплата** — картка або готівка, автоматичний підрахунок решти, автоматичне списання залишків.
- **Товари** (`/products`) — CRUD товарів: фото, назва, артикул (SKU), штрих-код, категорія,
  ціна, залишок, мінімальний залишок. Пошук, фільтри, експорт у CSV.
- **Склад** (`/warehouse`) — швидке коригування залишків (+/-), позначення товарів, що
  закінчуються, експорт у CSV.
- **Історія продажів** (`/history`) — фільтри за датою і типом оплати, пошук, деталізація чеків,
  експорт у CSV.
- **Статистика** (`/stats`) — виручка за сьогодні / 7 днів / 30 днів, розподіл готівка/картка,
  графік по днях, топ товарів, товари з малим залишком.
- **Авторизація** — проста автентифікація адміністратора (логін/пароль), сесія у httpOnly cookie.

## Технології

Next.js 14 (App Router), React 18, TypeScript, Tailwind CSS, Prisma ORM, PostgreSQL, JWT-сесії
(бібліотека `jose`), bcrypt для паролів.

## Встановлення

### 1. Передумови

- Node.js 18+ 
- PostgreSQL 14+ (локально, у Docker, або хмарний — напр. Neon, Supabase, Railway)

### 2. Встановіть залежності

```bash
npm install
```

Команда `postinstall` автоматично викличе `prisma generate`.

### 3. Налаштуйте змінні середовища

Скопіюйте `.env.example` у `.env` і заповніть:

```bash
cp .env.example .env
```

```env
DATABASE_URL="postgresql://user:password@localhost:5432/pos_db?schema=public"
SESSION_SECRET="довгий-випадковий-рядок"   # згенеруйте: openssl rand -base64 32
ADMIN_USERNAME="admin"
ADMIN_PASSWORD="ваш-пароль"
```

### 4. Створіть таблиці в базі даних

```bash
npx prisma migrate dev --name init
```

### 5. Заповніть базу тестовими даними (опційно, але зручно для першого запуску)

```bash
npm run db:seed
```

Це створить адміністратора (з `.env`) та кілька демо-товарів.

### 6. Запустіть додаток

```bash
npm run dev
```

Відкрийте http://localhost:3000 і увійдіть під логіном/паролем з `.env`.

### 7. Продакшн-збірка

```bash
npm run build
npm run start
```

## Встановлення як застосунок на телефон (PWA)

Відкрийте сайт у Chrome на Android (або Safari на iOS) → меню браузера → "Додати на головний
екран" / "Install app". Додаток отримає власну іконку і відкриватиметься на весь екран, без
адресного рядка.

> Примітка: PWA тут забезпечує зручну інсталяцію та іконку на головному екрані. Оскільки додаток
> постійно працює з базою даних (продажі, залишки), повноцінна офлайн-робота не передбачена —
> для оформлення продажів потрібне інтернет-з'єднання.

## Сканування штрих-кодів

Використовується нативний браузерний **BarcodeDetector API** через камеру телефону — без
додаткових залежностей. Він добре працює в Chrome на Android. Якщо браузер не підтримує цей
API (наприклад, Safari на iOS), додаток показує поле для ручного вводу коду — туди ж
чудово вводить дані звичайний USB/Bluetooth-сканер, який працює як клавіатура.

Якщо потрібна ширша підтримка сканування "з коробки" на всіх пристроях, можна підключити
бібліотеку `html5-qrcode` або `zxing-js` та замінити реалізацію у
`src/components/BarcodeScanner.tsx`.

## Структура проєкту

```
prisma/
  schema.prisma       — модель даних (Product, Category, Sale, SaleItem, Admin)
  seed.ts             — наповнення демо-даними
src/
  app/
    page.tsx           — екран продажу (POS)
    login/             — сторінка входу
    products/          — менеджмент товарів
    warehouse/         — склад
    history/           — історія продажів
    stats/             — статистика
    api/               — REST API маршрути (products, sales, stats, auth, categories)
  components/          — Nav, BarcodeScanner, ProductImage, pos/*, stats/*
  lib/                 — prisma-клієнт, сесії (JWT), CSV-хелпер
  middleware.ts        — захист сторінок/маршрутів авторизацією
public/
  manifest.json, sw.js, icons/  — PWA
  uploads/             — завантажені фото товарів (локальне сховище)
```

## Зберігання фото товарів

Фото завантажуються на сервер і зберігаються у `public/uploads/`. Для production-деплою на
платформах з ефемерною файловою системою (Vercel тощо) рекомендується перенести завантаження на
об'єктне сховище (S3, Cloudflare R2, Supabase Storage) — місце для заміни: маршрут
`src/app/api/products/upload/route.ts`.

## Безпека

- Паролі зберігаються як bcrypt-хеші.
- Сесія — підписаний JWT у httpOnly, sameSite=lax cookie (30 днів).
- Усі сторінки та API, крім `/login`, захищені middleware.
- Це "проста авторизація адміністратора" — один обліковий запис. Для кількох ролей
  (адміністратор/касир) модель `Admin` легко розширити полем `role`.
